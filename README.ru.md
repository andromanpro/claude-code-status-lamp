# Claude Code Status Lamp 🚦

[English](README.md) · **Русский**

![Лампа-индикатор статуса Claude Code: синий → янтарный → зелёный](docs/lamp-colors.gif)

**Бросить взгляд на лампу вместо того, чтобы сторожить терминал.** Превращает [WLED](https://github.com/wled/WLED)-лампу — в том числе перепрошитую [GyverLamp](https://github.com/AlexGyver/GyverLamp) — в окружающий индикатор статуса для [Claude Code](https://claude.com/claude-code).

Настоящая фишка не «синий = работает». А **янтарный = ты нужен Claude**. Уходишь в свою задачу, отводишь взгляд от экрана — и лампа сама позовёт, как только Claude упрётся в разрешение или вопрос. Если ты склонен к гиперфокусу, в этом весь смысл: вернуть твоё внимание, не заставляя пялиться в терминал в ожидании.

- 🔵 **Синий** — Claude работает
- 🟡 **Янтарный** — нужен ты (разрешение или ждёт твой ввод)
- 🟢 **Зелёный** — простой, твой ход
- 🔴 **Красный** — ход упал

Без поллинга и без обязательного фонового сервиса — только крошечный скрипт на стандартной библиотеке Python, который шлёт один HTTP-запрос лампе из хуков жизненного цикла Claude Code. Он **fail-silent**: лампа офлайн или в другой сети → хук молча ничего не делает и не блокирует Claude. *(Опциональный [фоновый демон](#фоновый-демон-опционально) переоценивает состояние по таймеру — полезно, если `idle_prompt` доходит не всегда — и служит основой для зон по агентам. Схеме «только хуки» ниже демон не нужен вовсе.)*

## Как это работает

Хуки Claude Code ([hooks](https://code.claude.com/docs/en/hooks)) запускают shell-команду на событиях жизненного цикла. Каждое событие зовёт `lamp_status.py <состояние>`, который POST-ит `{"ps": N}` в JSON-API WLED и применяет пресет:

```
UserPromptSubmit, PreToolUse       ──▶ working    ──▶ пресет 1 (синий)
Notification [permission_prompt]   ──▶ attention  ──▶ пресет 2 (янтарный)
Notification [idle_prompt]         ──▶ done       ──▶ пресет 3 (зелёный)
SessionStart                       ──▶ done       ──▶ пресет 3 (зелёный, покой)
StopFailure                        ──▶ error      ──▶ пресет 5 (красный)
SessionEnd                         ──▶ off
```

**Почему `Stop` намеренно НЕ используется.** `Stop` срабатывает в конце любого хода — в том числе когда Claude запускает фонового субагента/воркфлоу и *всё ещё работает*. Связка `Stop → зелёный` красит лампу «готово» посреди работы (ложный зелёный). Поэтому зелёный приходит от `idle_prompt` (Claude реально простаивает, ждёт тебя) и от `SessionStart`.

Компромисс: **зелёный появляется через ~60 с после конца хода, а не мгновенно**, потому что `idle_prompt` срабатывает после небольшого простоя. Это намеренно — мы меняем «мгновенный зелёный» на «зелёный, который никогда не врёт». (Не возвращай `Stop → done` «из лучших побуждений» — это вернёт ложный зелёный.)

## Требования

- **Python 3.6+** — только стандартная библиотека, никакого `pip install`. На Windows используй `py`, на macOS/Linux — `python3`.
- **WLED 0.14+** (проверено на 16.x). Старые сборки могут не принимать API сохранения пресетов.
- WLED-лампа — любой ESP8266/ESP32 + WS2812B, на который можно прошить WLED — в твоей локальной сети.

## Железо

- Любая плата ESP8266/ESP32 с лентой/матрицей WS2812B (или совместимой), прошитая **WLED**.
- **GyverLamp** (адресная матрица AlexGyver) отлично подходит — просто перепрошей её со стоковой прошивки на WLED.
- Нужно ≥4 МБ флеша (безопасный минимум; сборки на 1 МБ для ESP-01 есть, но тесные) и сеть **2.4 ГГц** в той же подсети, что и машина с Claude Code.

## Настройка

### 1. Прошить WLED

Браузерный флешер **[install.wled.me](https://install.wled.me/)** (Chrome/Edge). Выбери сборку под свой чип (ESP8266 или ESP32), поставь галку **Erase**, установи. Arduino IDE не нужен.

> **Перепрошиваешь GyverLamp?** Подключайся к **собственному** micro-USB платы контроллера (внешний разъём корпуса часто только питание), бери **дата**-кабель (не «только зарядка»), и поставь драйвер **CP2102 / CH340**, если COM-порт не появляется.

### 2. Подключить лампу к Wi-Fi 2.4 ГГц

Подключись к сети `WLED-AP` (пароль `wled1234`); страница настройки обычно открывается сама — если нет, зайди на `http://4.3.2.1`. Введи свой домашний Wi-Fi.

> ⚠️ **Грабли №1.** ESP8266 работает **только на 2.4 ГГц** и часто не цепляется к точке **Wi-Fi 6 (802.11ax)**. Если лампа не подключается:
> - выбери **2.4-ГГц** SSID (не `*_5G`);
> - в роутере переведи диапазон 2.4 ГГц в **802.11 b/g/n** — ищи тумблер **«отключить Wi-Fi 6 / 802.11ax»**, **«только b/g/n»** или **«совместимость»**. У разных вендоров он называется по-разному: **Xiaomi/Redmi** — *Wi-Fi settings → «WiFi 5 compatibility mode»*; **TP-Link/ASUS/Keenetic** — *Wireless mode → b/g/n mixed*;
> - шифрование держи **WPA2-PSK** (WPA3 ESP8266 не умеет).
>
> Это немного ослабляет безопасность этого диапазона — для домашней сети нормально (см. [Безопасность](#безопасность)).

### 3. Найти IP лампы

Глянь список клиентов роутера, попробуй `http://wled.local` или просканируй подсеть. Затем закрепи адрес **DHCP-резервом**, чтобы он не менялся.

### 4. Создать пресеты статусов

Запусти init-скрипт (подставь IP своей лампы):

```bash
python init_presets.py 192.168.1.50
```

Он запишет пресеты **1/2/3/5** и проверит каждый. ⚠️ Он перезаписывает эти слоты — поменяй номера в скрипте, если они уже заняты.

…либо восстанови [`presets.json`](presets.json) в WLED (Config → Security & Backup → Restore presets) — более надёжный путь на медленном ESP8266. В нём ещё есть необязательный пресет **4 «Ambient»** (радужная сцена, хуками не используется — оставь как фоновое свечение или игнорируй).

### 5. Выставить число диодов

WLED → **Config → LED Preferences** → задай число диодов под свою матрицу (например, **256** для 16×16) и пин данных (у GyverLamp это **GPIO2 / D4** — **проверь пин своей платы**). Пока не задал число — горит только часть матрицы, это нормально, а не поломка. Лимит тока выставь под свой блок питания.

### 6. Установить скрипт + хуки

- Положи `lamp_status.py` в стабильное место. IP лампы задай через переменную окружения `WLED_IP` или правкой `LAMP_IP` сверху файла.
- Домержи хуки из [`examples/settings-hooks.json`](examples/settings-hooks.json) в свой `~/.claude/settings.json`. **Домержи** в существующий объект `hooks`, не затирай его. Замени `<python>` на свой лаунчер (**Windows `py`**, **macOS/Linux `python3`**) и путь.
- Что означают поля: **`matcher`** фильтрует, какое под-событие запускает хук (для `Notification`: `permission_prompt` vs `idle_prompt`; для `PreToolUse`: имя инструмента, `""` = все); **`async: true`** запускает команду не блокируя ход; **`timeout`** ограничивает её по секундам.

### 7. Проверить

Отправь промпт → 🔵 синий. Подожди простой ~60 с → 🟢 зелёный. Вызови разрешение → 🟡 янтарный.

Ничего не происходит? Запусти `/hooks` (или перезапусти Claude Code), чтобы перечитать `settings.json`, и продебажь скрипт руками:

```bash
WLED_DEBUG=1 python lamp_status.py working
```

`WLED_DEBUG=1` печатает трейсбек (неверный IP, не та сеть, пресеты не созданы) вместо тихого выхода.

## Кастомизация

- **Цвета / яркость:** правь RGB и `bri` в `init_presets.py` (или пересохрани пресеты в приложении WLED). Синий горит ~95% времени, поэтому он самый тусклый; янтарный самый яркий, потому что именно он должен тебя дёрнуть. Для ночи убавь сильнее.
- **Анимированные статусы:** пресет WLED может хранить любой эффект, не только сплошной цвет — сделай «работаю» медленным «дыханием» синего, «внимание» — миганием. Просто пересохрани пресет с выбранным эффектом.
- **Больше состояний:** повесь другие хуки (например, `SubagentStop`, `PreCompact`) на свои пресеты.

## Несколько сессий

Запускай сколько угодно сессий Claude Code на одну лампу. Состояние каждой отслеживается по её `session_id` (читается из stdin хука), и лампа показывает **самое приоритетное** состояние среди всех живых сессий:

> внимание (янтарный) · ошибка (красный) · работаю (синий) · готово (зелёный)

Так **янтарный от ЛЮБОЙ сессии залипает**, пока та не разблокируется — занятая или простаивающая сессия не перекрасит «кто-то тебя зовёт». Когда заканчивается последняя сессия, лампа гаснет. Скрипт ещё и душит повторы, так что шквал событий `PreToolUse` «работаю» шлёт максимум один запрос.

## Фоновый демон (опционально)

Хуки применяют лампу на каждое событие, поэтому лампа переоценивается *только когда срабатывает хук*. Если сессия закончила работу, а `idle_prompt` не пришёл — или сессию убили без чистого `SessionEnd` — её состояние «работаю» может зависнуть и держать лампу синей, пока не сработает какой-нибудь другой хук.

[`lamp_daemon.py`](lamp_daemon.py) закрывает эту дыру. Это маленький always-on процесс: раз в несколько секунд (`LAMP_POLL`, дефолт 5с) перечитывает общий файл состояния, считает «работаю»-сессию, молчавшую дольше окна свежести, простаивающей, и применяет победивший пресет — так лампа сама сходит в зелёный/гаснет, без всякого хука. Хуки продолжают работать как раньше; демон — страховка сверху (и фундамент для зон по агентам). Переиспользует всё из `lamp_status.py` — без новых зависимостей.

Быстрая проверка, не трогая лампу: `python lamp_daemon.py --once --dry`.

**Windows (Task Scheduler, без админа).** Поправь путь к `pythonw.exe` в [`examples/install-daemon.ps1`](examples/install-daemon.ps1) и запусти:

```powershell
powershell -ExecutionPolicy Bypass -File examples\install-daemon.ps1
```

Он зарегистрирует `ClaudeLampDaemon` на старт при входе (авто-рестарт, без окна консоли). Управление — `schtasks /Run|/End|/Delete /TN ClaudeLampDaemon`.

**macOS/Linux.** Запусти `python3 lamp_daemon.py` под launchd / `systemd --user` / `@reboot` cron.

## Ограничения

- **Зелёный отстаёт на ~60 с** от реального конца хода — см. [Как это работает](#как-это-работает). Это намеренная цена за то, что зелёный никогда не врёт. (Опциональный [демон](#фоновый-демон-опционально) гарантирует, что лампа *в итоге* сойдёт в правильное состояние, даже если `idle_prompt` не пришёл, но зелёный от этого быстрее не появляется.)

## Безопасность

- **У WLED нет аутентификации в локальной сети.** Любой в той же Wi-Fi может управлять лампой (или перекрасить её). Держи её в доверенной домашней сети; никогда не пробрасывай порт 80 в интернет. При желании WLED умеет пароль на точку и PIN на настройки.
- **Хуки запускают произвольные локальные команды.** Ты домерживаешь в `~/.claude/settings.json` конфиг, исполняющий команды. Читай любой хук-конфиг (включая этот) перед добавлением и направляй его только на скрипты, которые сам прочитал.
- Шаг 2 ослабляет диапазон 2.4 ГГц (b/g/n, WPA2), чтобы поднять ESP8266 — небольшой даунгрейд этого диапазона. Дома норм; в общей/офисной сети подумай дважды.

## Решение проблем

| Симптом | Что делать |
|---|---|
| Лампа не цепляется к Wi-Fi | Только 2.4 ГГц **и** отключить Wi-Fi 6 на роутере (шаг 2) |
| Горит только часть матрицы | Выставить число диодов (шаг 5) |
| Нет COM-порта при прошивке | Поставить драйвер CP2102/CH340; взять **дата**-кабель; втыкать в USB **самой платы**, не в разъём питания корпуса |
| Хуки не срабатывают | `/hooks` (или перезапуск Claude Code) для перечитки настроек; продебажить `WLED_DEBUG=1 python lamp_status.py working` |
| Зелёный, когда Claude явно занят | Висит `Stop → done` — убери (см. «Почему Stop намеренно не используется») |
| Янтарный и зелёный не различаются | `Notification` нужны раздельные matcher'ы (`permission_prompt` vs `idle_prompt`), а не `matcher: ""` |
| IP сменился, лампа не реагирует | Поставить DHCP-резерв; обновить `WLED_IP` |
